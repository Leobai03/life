# SOP Template

Use this template to generate the user's personal life SOP. Keep it practical and stage-specific.

## Personal Life SOP v1

### Stage Diagnosis

```text
You are currently in [stage]. The main pattern is [diagnosis].
The goal of this version is [small, realistic goal].
```

### Stage Theme

```text
[Protect priority 1], [restore priority 2], and [build priority 3] without [cost to avoid].
```

### Protected Priorities

- Priority 1:
- Priority 2:
- Priority 3:

If the user only has 1-2 real priorities, leave it at 1-2. Do not invent a fake third priority.

### Daily Pass Line

Define 3-5 actions that make a day count as passed.

Good examples:

- Eat 2 meals.
- Complete 1 direct work or income action.
- Leave 1 content, learning, or reflection trace.
- Go outside or move for 20 minutes.
- Start shutdown by the target time.

### Morning Or Start Protocol

Use when the user wakes up or starts their day, even if it is not morning.

```text
1. Wash up.
2. Drink water.
3. Eat something simple.
4. Write today's one most important action.
5. Start before checking noisy feeds or optional messages.
```

### Main Work Block

Define one protected block. Keep it small enough to start.

Default:

```text
Work for 60-90 minutes on the action most directly tied to the current priority.
During this block, do not handle optional chats, feeds, or new ideas.
```

### Creation Or Learning Trace

The user does not need to publish daily. They need a trace.

Options:

- Draft 1 post.
- Record 1 real observation.
- Summarize 1 case.
- Learn 1 concept and write 3 lines.
- Capture 1 question worth exploring.

### Body Baseline

Define the minimum viable body routine.

Default:

- Eat at least 2 meals.
- Go outside or move for 15-30 minutes.
- Set a realistic sleep target.
- Do not use work urgency as an unlimited excuse to skip food or sleep.

### Evening Shutdown

Design this around the user's most unstable time.

Examples:

- After 23:30, avoid conversations that obviously create emotional spirals.
- After 00:30, do not make major decisions.
- If stuck in feeds or loops, leave the screen for 15 minutes.
- Write the daily check-in before bed.

### Emotional Emergency Protocol

Use this when the user starts self-attacking or spiraling.

```text
1. Stop naming yourself with a moral label.
2. Ask: Did I sleep, eat, move, or overload myself poorly today?
3. Pick one physical reset: water, shower, walk, meal, room reset.
4. Do the smallest next action for 10 minutes.
5. Resume the day from the pass line, not from the original ideal plan.
```

### Weekly Review

Use 5-7 questions:

- What action actually protected or improved my life this week?
- What looked productive but mostly drained me?
- Which day had the best state, and why?
- Which day was most unstable, and why?
- What rule should become lighter?
- What rule should become stricter?
- What 3 things will I keep next week?

### Boundaries

Write 3-7 boundaries for the user's current risks.

Examples:

- Do not use income fluctuation to decide self-worth.
- Do not let traffic or online feedback break sleep.
- Do not use planning as a way to avoid the first action.
- Do not turn every relationship into work.
- Do not chase every opportunity while the body baseline is broken.

### Review Date

Set a short review point:

```text
Run this for 7 or 14 days, then create v2 based on what actually happened.
```
